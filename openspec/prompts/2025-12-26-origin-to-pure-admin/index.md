# 2025-12-26 Origin 项目迁移到 Pure-Admin 方案

## 1. 概述

本文档记录了将 Origin 项目（原始 10WMS 项目）迁移到基于 vue-pure-admin 的 Main 项目的完整方案。

### 1.1 项目背景

- **Origin 项目**: 原始的 10WMS 项目，待改造的项目
- **Main 项目**: 基于 vue-pure-admin 重构的新项目
- **目标**: 在保留 Pure-Admin 模板优势的同时，完整迁移旧项目的业务代码和功能

### 1.2 迁移目标

1. **保留业务逻辑**: 完整迁移所有业务功能，不丢失任何功能
2. **提升用户体验**: 利用 Pure-Admin 的完善布局和主题系统
3. **提高开发效率**: 使用 Pure-Admin 的丰富组件和工具函数
4. **改善可维护性**: 统一代码规范和架构设计
5. **优化性能**: 利用 Pure-Admin 的性能优化方案

### 1.3 预期收益

- ✅ 更好的用户体验（完善的布局、主题、权限系统）
- ✅ 更高的开发效率（丰富的组件库和工具函数）
- ✅ 更好的可维护性（统一的代码规范和架构）
- ✅ 更好的性能（完善的性能优化方案）
- ✅ 更好的扩展性（插件机制、主题定制）

## 3. 迁移概览

### 3.1 技术栈对比

|    技术项    |           Origin            |        Pure-Admin        | 迁移难度  |
| :----------: | :-------------------------: | :----------------------: | :-------: |
|     Vue      |           3.5.26            |          3.5.26          | ✅ 无差异 |
|  Vue Router  |            4.6.4            |          4.6.4           | ✅ 无差异 |
|    Pinia     |            3.0.4            |          3.0.4           | ✅ 无差异 |
| Element Plus |           2.13.0            |          2.13.0          | ✅ 无差异 |
|   样式方案   |          CSS/SCSS           |      Tailwind/SCSS       | ⚠️ 需适配 |
|  状态持久化  | pinia-plugin-persistedstate |    responsive-storage    | ⚠️ 需适配 |
|  HTTP 工具   |            axios            | axios + @pureadmin/utils | ⚠️ 需适配 |

### 3.2 业务模块清单

Origin 项目包含 13 个核心业务模块：

1. **基础配置** - 自动编码、编码类型、计量单位等
2. **基础数据** - 商品、客户、供应商管理
3. **采购管理** - 预约采购、批量收货、库存查询
4. **库存管理** - 综合盘点、移库盘点、货架调整
5. **出库管理** - 拣货管理、出库类型
6. **日常检查** - 异常发货、温度维护
7. **计费配置** - 计费模式、费用模板
8. **系统管理** - 用户、角色、菜单、权限
9. **消息中间件** - 消息中心、消息模板
10. **区域配置** - 区域信息、城市类型
11. **人员配置** - 学历代码、就业状态
12. **仓库配置** - 订单类型
13. **客户报表** - 库存报表、有效期预警

### 3.3 迁移阶段

#### 阶段一：基础设施迁移（1-2 周）

- HTTP 请求层适配
- 状态管理适配
- 路由系统适配
- 工具函数迁移

#### 阶段二：公共组件迁移（2-3 周）

- 基础组件（表格、表单、分页等）
- 业务组件（表格工具栏、搜索组件等）
- 特殊功能组件（表单设计器、流程图等）

#### 阶段三：API 层迁移（1-2 周）

- API 结构调整
- API 适配
- 统一响应处理

#### 阶段四：业务模块迁移（6-8 周）

- P0: 系统管理、登录认证（2 周）
- P1: 基础数据、采购管理、库存管理（2 周）
- P2: 出库管理、日常检查、基础配置（2 周）
- P3: 其他辅助模块（2 周）

#### 阶段五：测试与优化（2-3 周）

- 功能测试
- 性能优化
- 用户体验优化

**总计**: 12-16 周

## 5. 关键决策

### 5.1 使用 Pure-Admin 的功能

**建议使用**:

- ✅ 权限系统（完善的按钮级权限控制）
- ✅ 布局系统（多种布局模式、标签页管理）
- ✅ 主题系统（多主题切换、暗色模式）
- ✅ `@pureadmin/table`（功能更强大的表格组件）
- ✅ `@pureadmin/utils`（丰富的工具函数）
- ✅ `responsive-storage`（响应式存储）
- ✅ Tailwind CSS（现代化的样式方案）

### 5.2 保留 Origin 的功能

**建议保留**:

- ✅ 表单设计器（`@form-create/designer`）
- ✅ 打印功能（`vue-plugin-hiprint`）
- ✅ 自定义业务组件（`base-form`、`dinamic-table-form` 等）
- ✅ 业务逻辑（完整保留，不重写）

### 5.3 样式方案

**建议**:

- 主要使用 Tailwind CSS（原子化 CSS，开发效率高）
- 保留必要的 SCSS 样式（复杂样式、动画等）
- 统一使用 Pure-Admin 的主题系统
