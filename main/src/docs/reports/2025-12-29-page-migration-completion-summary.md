## 5. 遇到的问题和解决方案

### 5.1 类型兼容性问题

**问题**：`menuType` 和 `RouteRecordRaw` 类型不兼容

**解决方案**：

- 定义通用的 `RouteMenuItem` 类型
- 使用泛型 `<T extends RouteMenuItem>` 支持多种类型
- 保持类型推断，输入什么类型返回什么类型

### 5.2 路由归属混乱

**问题**：业务路由和框架示例路由混在一起，难以区分

**解决方案**：

- 将路由分为 `business` 和 `pure-admin` 两个目录
- 实现侧边栏类型区分功能
- 支持约定式配置和显式配置

### 5.3 页面目录不统一

**问题**：业务页面分散在 `pages` 和 `views` 目录

**解决方案**：

- 统一将业务页面放在 `pages` 目录
- `views` 目录只保留框架示例页面
- 更新所有路由配置指向正确的目录

### 5.4 API 调用方式不统一

**问题**：Origin 项目使用回调方式，Pure-Admin 使用 Promise 方式

**解决方案**：

- 统一使用 Promise/async-await 方式
- 创建统一的 HTTP 工具类
- 实现统一的错误处理

### 5.5 样式冲突

**问题**：Origin 项目和 Pure-Admin 的样式系统不同

**解决方案**：

- 统一使用 Tailwind CSS
- 保留 Element Plus 组件样式
- 适配 Pure-Admin 的主题系统
