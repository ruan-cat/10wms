### 4.2 配置规范

1. **国际化**: 所有路由标题使用 `$t("business.xxx")` 引用 i18n 键
2. **图标**: 优先使用细线风格图标（如 `ri:xxx-line`）
3. **懒加载**: 所有组件使用动态导入
4. **类型安全**: 使用 `satisfies RouteConfigsTable` 确保类型正确

## 7. 遇到的问题和解决方案

### 7.1 路由配置问题

**问题**: 初期使用了自动路由配置（vue-router/auto），导致路由管理复杂且不透明。

**解决方案**:

- 移除自动路由依赖
- 采用 Pure-Admin 标准的手动路由配置
- 实现路由分层管理（pure-admin 和 business 目录）

### 7.2 状态持久化问题

**问题**: Origin 项目使用 pinia-plugin-persistedstate，Pure-Admin 使用 responsive-storage。

**解决方案**:

- 适配 responsive-storage API
- 保持状态数据结构一致
- 实现等效的持久化逻辑

### 7.3 组件样式适配问题

**问题**: Origin 项目的组件样式需要适配 Pure-Admin 的主题系统。

**解决方案**:

- 使用 Tailwind CSS 原子类
- 保留必要的 SCSS 模块
- 确保组件在亮色/暗色主题下正常显示

### 7.4 TypeScript 类型问题

**问题**: 部分页面存在 TypeScript 类型错误。

**解决方案**:

- 修复表格组件使用方式
- 完善类型定义
- 使用 getDiagnostics 工具检查类型错误
