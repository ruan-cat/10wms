## 2. 图标风格偏好

### 2.1 核心原则

1. **细线风格优先**：优先使用细线条图标，避免实心图标
2. **图标集选择**：主要使用 Remix Icon (`ri:`) 图标集
3. **语义清晰**：图标应与功能语义匹配，易于识别
4. **风格统一**：同一模块内的图标风格保持一致

### 2.2 具体要求

**推荐做法**：

- ✅ 使用 Remix Icon 的 `-line` 后缀图标（细线风格）
- ✅ 使用 Solar Icon Set 的 `-linear` 后缀图标（细线风格）
- ✅ 图标与功能语义匹配
- ✅ 父级和子级路由都配置图标

**避免做法**：

- ❌ 使用 `-fill` 后缀图标（实心风格）
- ❌ 使用 `-solid` 后缀图标（实心风格）
- ❌ 图标语义不清晰或与功能不匹配
- ❌ 同一模块内图标风格不统一

## 7. 使用示例

### 7.1 完整路由配置

```typescript
import { $t } from "@/plugins/i18n";

const Layout = () => import("@/layout/index.vue");

const systemRouter: RouteConfigsTable = {
	path: "/system",
	name: "System",
	component: Layout,
	redirect: "/system/user",
	meta: {
		title: "系统管理",
		icon: "ri:shield-user-line", // 父级图标：细线风格
		rank: 10,
	},
	children: [
		{
			path: "/system/user",
			name: "User",
			component: () => import("@/pages/system/user/index.vue"),
			meta: {
				title: "用户管理",
				icon: "ri:user-line", // 子级图标：细线风格
			},
		},
		{
			path: "/system/role",
			name: "Role",
			component: () => import("@/pages/system/role/index.vue"),
			meta: {
				title: "角色管理",
				icon: "ri:admin-line", // 子级图标：细线风格
			},
		},
	],
};

export default systemRouter;
```

## 8. 最佳实践

### 8.1 图标命名规范

- **使用语义化名称**：选择能够清晰表达功能的图标
- **保持一致性**：同类功能使用相似风格的图标
- **避免歧义**：不使用可能产生多种理解的图标

### 8.2 图标大小和间距

Pure-Admin 会自动处理图标的大小和间距：

- **菜单图标**：自动调整为合适大小
- **图标间距**：自动添加与文字的间距
- **响应式**：在不同屏幕尺寸下自动适配

### 8.3 图标颜色

图标颜色由主题系统控制：

- **默认状态**：使用主题的次要文字颜色
- **激活状态**：使用主题的主色调
- **悬停状态**：颜色略微加深
- **暗色模式**：自动适配暗色主题

## 10. 常见问题

### 10.1 为什么优先使用细线图标？

**原因**：

1. **视觉轻盈**：细线图标更加简洁优雅
2. **现代感强**：符合当前设计趋势
3. **易于识别**：在小尺寸下仍然清晰
4. **主题适配**：在亮色和暗色主题下都表现良好

### 10.2 什么情况下可以使用实心图标？

**建议**：

- 尽量避免使用实心图标
- 如果必须使用，应该：
  - 仅在特殊强调的场景使用
  - 确保与整体风格协调
  - 征得团队同意

### 10.3 如何选择合适的图标？

**步骤**：

1. 理解功能含义
2. 搜索相关关键词
3. 对比多个候选图标
4. 选择语义最匹配的
5. 在实际场景中预览

### 10.4 图标不显示怎么办？

**排查步骤**：

1. 检查图标名称是否正确
2. 检查图标前缀是否正确（`ri:`）
3. 检查网络连接（Iconify 需要联网加载）
4. 查看浏览器控制台是否有错误
5. 尝试使用其他图标验证
