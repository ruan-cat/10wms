## 5. 迁移标准和规范

### 5.1 组件替换标准

|         原组件          |     新组件      |  迁移方式  |
| :---------------------: | :-------------: | :--------: |
|     ComponentsTable     | SimpleDataTable |  完全替换  |
| ComponentsDialogPromise |    el-dialog    |  完全替换  |
|       Pagination        |  el-pagination  |  完全替换  |
|        base-form        |    BaseForm     | 保留并适配 |

### 5.2 API 调用转换

```typescript
// 原方式（回调）
listAPI(params, successCallback, errorCallback);

// 新方式（Promise）
const response = await http.request({
	url: "/api/xxx",
	method: "post",
	data: params,
});
```

### 5.3 路由配置标准

```typescript
{
  path: "/module",
  name: "Module",
  component: Layout,  // 必须使用 Layout
  redirect: "/module/page",
  meta: {
    title: $t("menus.module"),
    icon: "ep:menu",
    rank: 10,
  },
  children: [
    {
      path: "/module/page",
      name: "ModulePage",
      component: () => import("@/pages/module/page/index.vue"),  // 懒加载
      meta: {
        title: $t("menus.page"),
        showLink: true,
      },
    },
  ],
}
```

### 5.4 代码规范标准

1. **命名规范**: kebab-case (文件), camelCase (变量), PascalCase (类型)
2. **注释规范**: JSDoc 格式
3. **类型规范**: 完整的 TypeScript 类型
4. **格式规范**: Prettier 统一格式化
5. **提交规范**: Commitlint 规范提交信息
