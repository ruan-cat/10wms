# Change: Origin 项目迁移到 Pure-Admin

## Why

将 Origin 项目（原始 10WMS 项目）完整迁移到基于 vue-pure-admin 的 Main 项目，以提升用户体验、开发效率和代码可维护性。Origin 项目虽然功能完整，但缺乏现代化的权限系统、主题系统和布局系统。Pure-Admin 提供了完善的后台管理基础设施，可以大幅提升项目质量。

## What Changes

### 阶段一：基础设施迁移（1-2 周）
- HTTP 请求层适配：将 Origin 的 axios 配置迁移到 Pure-Admin 的 HTTP 工具
- 状态管理适配：将 pinia-plugin-persistedstate 替换为 responsive-storage
- 路由系统适配：集成 Pure-Admin 的权限系统和路由守卫
- 工具函数迁移：将 composables 迁移到 utils 目录

### 阶段二：公共组件迁移（2-3 周）
- 基础组件迁移：表格、表单、分页、对话框等
- 业务组件迁移：表格工具栏、搜索组件等
- 特殊功能组件：保留并适配表单设计器、流程图、打印功能

### 阶段三：API 层迁移（1-2 周）
- API 结构调整：从 `apis/` 迁移到 `api/`
- API 适配：适配 Pure-Admin 的 HTTP 请求工具
- 统一响应处理：建立统一的响应数据结构

### 阶段四：业务模块迁移（6-8 周）
- P0（2 周）：系统管理、登录认证
- P1（2 周）：基础数据、采购管理、库存管理
- P2（2 周）：出库管理、日常检查、基础配置
- P3（2 周）：其他辅助模块（计费配置、消息中间件、区域配置、人员配置、仓库配置、客户报表）

### 阶段五：测试与优化（2-3 周）
- 功能测试：单元测试、集成测试、E2E 测试
- 性能优化：路由懒加载、组件按需加载、打包体积优化
- 用户体验优化：响应式适配、加载状态优化、错误提示优化

## Impact

### 影响的规范
- `infrastructure`: 基础设施层（HTTP、状态管理、路由）
- `components`: 组件层（公共组件、业务组件）
- `api`: API 层（接口定义、请求处理）
- `business-modules`: 业务模块层（13 个核心业务模块）
- `testing`: 测试层（单元测试、集成测试、E2E 测试）

### 影响的代码
- `main/src/utils/http/`: HTTP 请求工具
- `main/src/store/`: 状态管理
- `main/src/router/`: 路由系统
- `main/src/components/`: 公共组件
- `main/src/api/`: API 接口
- `main/src/views/`: 业务页面

### 风险与应对
- **功能遗漏（高）**: 建立功能清单，逐一核对
- **API 接口变更（高）**: 建立 API 文档，统一接口规范
- **依赖版本冲突（中）**: 提前检查依赖，使用 pnpm overrides
- **样式兼容性（中）**: 建立样式测试用例
- **工期延误（中）**: 预留缓冲时间，及时调整计划

### 预期时间
12-16 周（包含所有阶段）

### 建议团队规模
2-3 人
